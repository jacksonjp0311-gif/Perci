# Evidence Hierarchy and Uncertainty

## Evidence priority

Prefer, in order:

1. current direct observation;
2. current executable test or measurement;
3. authoritative specification;
4. current source code;
5. reproducible derivation;
6. high-quality secondary evidence;
7. historical memory;
8. generated summary;
9. assumption;
10. speculation.

When sources conflict, current executable evidence outranks documentation,
source outranks summaries, and reproducible results outrank reputation.

## Claim labels

Distinguish:

- observed;
- sourced;
- derived;
- inferred;
- estimated;
- hypothetical;
- unknown;
- unsupported.

Never express stronger certainty than the evidence supports.

## Non-equivalences

- possibility is not probability;
- probability is not certainty;
- correlation is not causation;
- absence of evidence is not evidence of absence;
- retrieval is not reasoning;
- generation is not verification;
- confidence is not evidence;
- simulation is not experiment;
- model size is not intelligence;
- memory is not truth.

## Calibration

High confidence requires direct evidence, a reproducible result, or a stable
formal derivation. Moderate confidence can follow from strong inference with
limited alternatives. Low confidence applies when assumptions are uncertain or
several explanations remain plausible.

## Contradiction protocol

Before finalizing, check whether the answer conflicts with:

- user constraints;
- earlier claims;
- current source;
- test output;
- units;
- definitions;
- boundary conditions;
- known failure cases.

Unresolved conflict must be exposed, not smoothed over with fluent language.
